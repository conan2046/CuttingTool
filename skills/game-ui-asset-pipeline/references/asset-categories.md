## 分类规则

### Panel

- 生成完整面板或边框，不生成界面正文。
- 正面平视，避免透视和旋转。
- 中央内容区保持干净。
- 角饰和边框不得跨槽位。
- 九宫格边界当前只记录人工配置，不自动推断。

### Button

- 默认把同一按钮的状态放在同一行。
- 推荐状态顺序：`Normal`、`Pressed`、`Disabled`。
- 需要 `Selected` 或 `Hover` 时扩充列数或单独生成。
- 状态之间必须保持相同轮廓、尺寸和装饰结构。
- 不生成按钮文字，文字由引擎运行时承载。

### Icon_Nav 与 Icon_General

- 保持统一外轮廓重量和观察角度。
- 缩小到目标尺寸时仍能凭轮廓辨认。
- 不烘焙功能名称、数字或通知角标。

### Icon_Item 与 Icon_Equip

- 默认不生成品质框、数量和等级标记。
- 道具/装备主体与品质框分离，便于运行时组合。
- 一张 Sheet 使用统一观察角度和光源。

### Icon_Skill

- 默认把技能主体和圆形/方形底框分开。
- 允许紧贴主体的硬边能量，但不得出现脱离主体的散点或跨槽位光效。
- 如果必须保留大范围半透明光效，应拆成独立任务并进入人工 QA。

### Icon_Status

- 货币、体力、生命、魔法等状态资源按视觉族分组。
- 数值文本、进度数值和等级文本不得烘焙进图片。

### Icon_Effect

- V1 只处理色键可稳定提取的简单硬边特效。
- 烟雾、玻璃、柔光、液体和大面积半透明效果标记为高风险。
- 包含绿色时不要使用绿色色键；包含品红时不要使用品红色键。

## 拆分规则

- 单张图的资源数量不能超过布局容量。
- 超量时按稳定顺序拆成 `sheet-01`、`sheet-02`。
- 不同类别不能为了减少生成次数而混在同一张正式 Sheet。
- 强关联状态组可以共同生成，以保证轮廓一致。
