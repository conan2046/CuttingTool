# GPT Image 资源生成协议

## 两阶段生成

### 阶段一：风格基准图

风格基准图用于锁定：材质、色彩、描边、光照、细节密度、观察角度和移动端可读性。

它可以展示少量典型面板、按钮和图标，但不作为正式切割输入。

### 阶段二：分类资源 Sheet

每张生产 Sheet 只生成一个类别或一个强关联状态组，并同时使用：

- Canonical UI Style Reference：视觉身份参考。
- Layout Guide：槽位数量、间距、中心和安全区参考。

## 通用提示词骨架

```text
任务：生成游戏 UI 可自动拆分的【资源分类】资源 Sheet。

输入图片：
- 图1：Canonical UI Style Reference，只用于锁定风格、材质、色彩、描边、光照和美术品质。
- 图2：Layout Guide，只用于规定槽位数量、间距、居中和安全边距，不得复制引导线、标签、颜色或中心点。

资源清单，按从左到右、从上到下：
【有序资源列表】

布局：
- 画布：【宽】×【高】
- 【列】列×【行】行
- 共【数量】个资源
- 每个不可见槽位只有一个完整资源
- 每个资源完整、居中、互不接触
- 资源不得跨槽位、重叠或接触画布边缘

背景：
- 完全一致的纯色【色键】
- 无渐变、纹理、噪点、阴影、地面或光照变化
- 资源本体不得使用【色键】及近似色

禁止：
- 文字、数字、标签、编号、Logo、水印
- 棋盘格、可见网格、分隔线、边框辅助线
- 场景、地面、投影、倒影
- 槽位之间的装饰
- 裁切、重叠、空槽和额外资源

输出前自检：准确数量、正确顺序、完整分离、风格一致。
```

## 提示词构建顺序

按以下顺序组织，避免模型先关注装饰而忽略生产约束：

1. 任务类型和用途。
2. 输入图片角色。
3. 有序资源清单。
4. 布局和数量。
5. 风格、材质和视角。
6. 背景和色键。
7. 禁止项。
8. 输出前自检。

## 状态组约束

按钮等状态资源必须强调：

- 相同轮廓。
- 相同尺寸。
- 相同装饰结构。
- 相同观察角度。
- 只改变状态需要的亮度、压下感、饱和度或高光。

不要让每个状态变成不同款式。

## Panel 九宫格生成约束

Panel 预计进入 Unity 九宫格时，提示词必须额外明确：

- 独特装饰只放在四角固定区。
- 上、下、左、右四条边的中段保持连续、均匀、可重复拉伸。
- 中央内容区保持干净，不放徽记、光点、纹章、裂纹中心、文字或方向性纹理。
- 禁止在边中点生成星点、菱形、莲花、徽章、龙头或其他不可重复图案。
- 四角装饰不得伸入预计的水平或垂直拉伸带。

输出后必须按九宫格预览检查四条拉伸带。若中段仍出现独特装饰，只针对该问题编辑或重生成；不得通过扩大 Border 把边中段装饰全部纳入固定区，以免固定区在小尺寸控件中挤压变形。

确定性 QA 对每个 Panel/Button 输出 `nine_slice_stretch_bands`，同时检查四边中间 60% 的外轮廓偏差和预乘 RGBA 内部纹理梯度；Panel 兼容保留 `panel_stretch_bands`。任一边出现超过容差的凸起、凹口或持续花纹时，分别写入 `panel-stretch-band-decoration` 或 `button-stretch-band-decoration` 并硬失败；Prompt 约束、自动检测和多尺寸 Sliced 预览必须同时存在。

## 色键约束

使用纯色色键，不使用棋盘格。背景必须是单一颜色，不能带：

- 渐变
- 纹理
- 反射
- 接触阴影
- 地面
- 光照变化

普通不含绿色的 UI 首选 `#00FF00`；资源包含绿色时改用 `#FF00FF`。

## 复杂半透明特效

烟雾、玻璃、液体和柔光默认设置 `transparency_mode=model-matte-derived`：先用内置 GPT Image 2 生成纯黑底彩色 Sheet，再将该图作为编辑目标生成像素对齐的灰度 Alpha Matte。黑色表示透明、白色表示不透明、连续灰度表示部分透明。

只有用户明确要求生成源直接携带 Alpha 时才使用 `native-alpha-required`。两种模式的字段、提示词、预检、来源标记和失败降级见 `native-alpha-contract.md`。

## 失败重试

每次只针对一个失败原因重试：

- 数量错误：强化准确数量和有序清单。
- 元素相连：扩大槽位间距并缩小安全区内容占比。
- 裁边：增加外边距和安全区。
- 网格被复制：明确 Layout Guide 只作为不可见施工参考。
- 风格漂移：加强 Canonical Reference 的身份约束。
- 状态外形不一致：强调同一母版轮廓，只改变状态表现。
- Panel/Button 拉伸带有装饰：只保留四角装饰，清除四边中段和中央独特图案，重建连续边线，并生成多尺寸 Sliced 预览复核。

不要在一次重试中同时改变风格、布局和资源清单。

V0.13 由 `orchestrate_ui_delivery.py` 把 QA 的最高优先级单一缺陷写入 `prompts/<job-id>-retry-<NN>.md` 和 `qa/regeneration-plan.json`。Codex 必须使用该 Prompt，只覆盖计划列出的输入文件，再续跑编排器。默认最多 3 个候选；候选分数只用于排序，任何 `fail` 都禁止正式交付。

跨 Sheet 风格评分使用正式透明资源的可见 RGBA 画像，组合调色板、平均色、亮度、饱和度和边缘密度，同时比较 Canonical 与其他 Sheet。`cross-sheet-style-drift` 必须绑定 Job；纠错 Prompt 只要求恢复 Canonical 的配色、材质、描边、对比度和光照，不得借机改变布局或资源语义。
