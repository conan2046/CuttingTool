# 色键阈值与碎片合并契约

## 自适应阈值诊断

对纯色色键 Sheet 统计画布边框像素到目标色键的 RGB 欧氏距离，记录 `p50`、`p90`、`p95`、`p99`、`p99.5`、最大值和分布宽度。

- 建议透明阈值覆盖 `p99.5` 并保留安全余量。
- 不透明阈值必须大于透明阈值，并为抗锯齿软边保留单一过渡带。
- `high` 或 `medium` 置信度且未发现近色主体风险时，允许自动采用。
- `low` 置信度、建议透明阈值过宽或主体大面积落入过渡带时，禁止自动采用。
- 硬边部分透明光带只有在稳定不透明核心像素不少于含糊过渡像素时允许自动采用，并记录 `stable_core_support_ratio`；否则继续判近色主体风险。
- `medium` 必须记录 `variable-chroma-border`，并继续执行最终残色 QA。

诊断结果写入色键报告的 `threshold_diagnostics`。不得仅因为扩大阈值后资源数量正确，就忽略近色主体损伤风险。

## 净边与缩放边界

- 邻近稳定前景投影默认作用于 `transparent_threshold` 与 `opaque_threshold` 之间的软遮罩过渡带。对背景邻域中符合“色键—局部前景”线性混合模型，或具有明显色键通道优势的像素，允许跨越固定欧氏距离阈值继续重建；这用于处理绿色主体叠品红色键等互补色边缘。
- 稳定前景种子必须排除具有色键通道优势的污染像素；种子距离门槛应覆盖蓝、白、金、绿等合法前景，不得只保留距离色键最远的颜色。
- 远离背景的确定前景必须保留原始 RGB 和 Alpha，禁止用局部模糊颜色覆盖主体内部纹理。
- 投影后仍无法安全重建、且颜色落在不透明阈值以内或具有色键通道偏色的部分 Alpha 残边应透明化，并记录 `discarded_unresolved_chroma_edge_pixels`。
- 大画布和放大后的 Sheet 可把局部前景搜索半径扩大到 `32px`，覆盖 10–20px 的模型色键反射带；投影仍受背景邻域约束。
- 背景邻域内仍不透明且具有强色键通道优势的像素应透明化，并记录 `discarded_opaque_chroma_spill_pixels`；禁止把该规则扩展到远离背景的确定前景。
- 只有 `near_key_subject_risk=false` 时允许丢弃上述无法解析残边；近色主体任务必须在更早阶段失败并转人工检查。
- 后续归一化必须采用预乘 Alpha 缩放。直通道 RGBA 插值会让低 Alpha 色键 RGB 在缩小时重新达到可见 Alpha，属于禁止做法。
- 每次净边或缩放算法变化都必须同时检查自动 QA 与 Contact Sheet；数量正确和脚本成功不能替代边缘视觉验收。
- 最终 QA 必须同时检查近纯色键距离和全部可见主体的色键通道优势。连续偏色超过容差时报告 `visible-chroma-spill`；主体内部出现强色键反射时应拒绝源图并重生成，禁止用全图去色破坏纹理。

## 抗锯齿失败矩阵

至少覆盖以下主体和背景组合：

| 主体 | 默认色键 | 通过条件 | 主要失败风险 |
|---|---|---|---|
| 金色 | 绿色 | 有真实部分 Alpha、无可见绿边 | 高光被误抠或绿边残留 |
| 白色 | 品红 | 白色中心保持不透明、无品红污染 | 高亮边缘发灰或残色 |
| 绿色 | 品红 | 绿色主体完整、无互补品红轮廓 | 欧氏距离过快变大导致混合边缘误判为前景 |
| 深色 | 绿色 | 暗部轮廓完整、软边不塌缩 | 暗边被当背景吞掉 |

同时保留三类失败样本：偏移色键背景应自适应通过；近色主体必须阻止自动采用；边框重污染必须降为低置信度并失败。另需覆盖复杂纹理内部不被投影漂白、绿色主体叠品红色键无可见轮廓、连续色键通道偏色被 QA 拦截，以及低 Alpha 色键像素不因归一化缩放重新显色。

P4 追加覆盖：硬边部分透明光带、白色高光叠金属边、深色主体叠深色描边、绿/品红双近色主体、大画布逐像素背景漂移、跨槽位噪声桥接，以及同一资源多个合法远离小装饰。

## 碎片分类

每个槽位先选择最大组件作为锚点，再按绝对距离和锚点包围框对角线比例迭代合并。

- `merged_component_count`：锚点及其安全近邻碎片数量。
- `detached_component_count`：未达到合并条件的远离组件数量。
- `major_detached_count`：面积达到主组件比例阈值的远离组件数量。

分类默认合并参数：

| 类别 | 比例 | 像素上限 |
|---|---:|---:|
| `Panel` | `0.06` | `64` |
| `Button` | `0.06` | `48` |
| `Icon_Nav` / `Icon_Status` / `Icon_General` / `Icon_Item` / `Icon_Equip` | `0.12` | `48` |
| `Icon_Skill` | `0.15` | `96` |
| `Icon_Effect` | `0.18` | `128` |

绝对最小合并距离默认 `12px`。实际允许间距为 `min(max(12, 对角线×比例), 像素上限)`，避免大面板按比例无限放宽，同时保留技能硬边火花等合法碎片。

近邻碎片可随主体导出且不产生警告。远离碎片默认产生 `detached-components`；存在显著第二主体时追加 `multiple-major-components`。请求可用 `fragment_policy` 覆盖分类参数。

连通域检测默认忽略 `alpha < 16` 的亚可见像素。一个连通组件同时覆盖两个或更多槽位中心时直接失败并报告 `cross-slot-connected-component`；合法伸入 gutter 但未跨过相邻槽位中心的主体继续允许。

`detached_action=allow-small` 只允许在同时设置正数 `small_detached_max_pixels` 与 `small_detached_max_anchor_ratio` 时使用。满足两项限制的小组件继续保留在导出图中，记录 `accepted_detached_count` 和 info，不产生 warning；不满足限制的组件仍 warning，显著第二主体永不因该配置放行。不得静默删除组件。
